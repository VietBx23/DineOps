# 📱 F&B Management System

## Multi-Store • Inventory • Role-Based Access (RBAC)

### 📘 Product Requirement Document (PRD) – Version 1.0

---

# 1. 📌 Overview

## 1.1 Purpose

Hệ thống F&B Management System được xây dựng nhằm:

* Quản lý chuỗi cửa hàng (multi-store)
* Hỗ trợ khách hàng order tại bàn qua QR
* Quản lý kho & nguyên vật liệu theo từng chi nhánh
* Phân quyền người dùng theo vai trò

---

## 1.2 Objectives

* Vận hành thực tế cho mô hình chuỗi F&B
* Đảm bảo dữ liệu tách biệt theo từng chi nhánh
* Tối ưu quy trình order → bếp → kho
* Là nền tảng mở rộng thành SaaS

---

## 1.3 Scope

### In Scope (MVP)

* QR Ordering tại bàn
* Multi-store (nhiều chi nhánh)
* Inventory cơ bản
* Role-based access control (RBAC)
* Realtime order (Socket)

### Out of Scope (Future)

* Thanh toán online
* Multi-warehouse
* AI prediction
* Báo cáo nâng cao

---

# 3. 👤 User & Role Management

## 3.1 Roles Definition

| Role    | Description           |
| ------- | --------------------- |
| admin   | Quản lý toàn hệ thống |
| manager | Quản lý 1 chi nhánh   |
| staff   | Nhân viên phục vụ     |
| kitchen | Nhân viên bếp         |

---

## 3.2 User Schema

```js
{
  _id,
  name,
  email,
  password,
  role,
  organizationId,
  storeId,
  status,
  createdAt
}
```

---

## 3.3 Authorization Rules (RBAC)

### Access Control Matrix

| Role    | Permissions         |
| ------- | ------------------- |
| admin   | Full access         |
| manager | Full within store   |
| staff   | Create order        |
| kitchen | View & update order |

---

## 3.4 Security Rules

* Mỗi user thuộc **1 store duy nhất**
* Không được truy cập data store khác
* Mọi API phải validate:

```js
req.user.storeId === resource.storeId
```

# 8. 🔄 Order → Inventory Integration

## 8.1 Auto Deduction Flow

Khi order chuyển sang trạng thái `cooking`:

1. Lấy danh sách nguyên liệu từ recipe
2. Tính tổng số lượng cần trừ
3. Trừ tồn kho
4. Log export transaction

---

## 8.2 Validation

* Nếu thiếu nguyên liệu:

  * Không cho chuyển cooking
  * Trả lỗi “Out of stock”

---

## 8.3 Transaction Requirement

* Phải dùng DB transaction
* Đảm bảo:

  * Không âm kho
  * Không lệch dữ liệu
